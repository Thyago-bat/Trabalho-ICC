Introdução

Este documento apresenta como funciona e como usar o sistema de controle de versões Git e o repositório GitHub.

Comandos Git

Exemplos de comandos Git:

  • init: Inicializa um novo repositório Git.
  • clone: Clona um repositório existente.
  • branch: Gerencia branches no repositório.
  • checkout: Muda para outra branch ou commit específico.
  • status: Mostra o estado atual do repositório.
  • diff: Mostra as diferenças entre commits.
  • add: Adiciona arquivos para o próximo commit.
  • commit: Grava as alterações no repositório.
  • push: Envia commits para o repositório remoto.
  • pull: Atualiza o repositório local com mudanças do repositório remoto.
  • revert: Reverte um commit específico.
  • merge: Une branches diferentes.
  • stash: Salva modificações temporariamente.
  • rm: Remove arquivos do repositório.
  • config: Configura as opções do Git.
  • reset: Reseta o repositório para um estado anterior.
  • remote: Gerencia repositórios remotos.
  • fetch: Baixa objetos e referências de outro repositório.
  • show: Mostra informações sobre objetos Git.
  • help: Mostra a ajuda para comandos Git.
  • rebase: Reaplica commits em cima de outro branch.
  • blame: Mostra a última modificação de cada linha de um arquivo.
  • tag: Gerencia tags no repositório.

GitHub

Configuramos e acessamos o repositório GitHub para armazenar e gerenciar nosso projeto. Utilizamos os comandos Git mencionados para adicionar, modificar e sincronizar os arquivos do projeto.

Veja abaixo alguns exemplos práticos de uso dos comandos Git:

Exemplo de Uso: Inicializar um Repositório

Para inicializar um novo repositório Git, usamos o comando git init:

git init

Exemplo de Uso: Clonar um Repositório

Para clonar um repositório existente, usamos o comando git clone:

git clone https://github.com/usuario/repo.git

Exemplo de Uso: Criar e Mudar de Branch

Para criar uma nova branch e mudar para ela, usamos os comandos git branch e git checkout:

git branch nova-branch
git checkout nova-branch

Exemplo de Uso: Adicionar e Commitar Arquivos

Para adicionar arquivos ao staging e fazer um commit, usamos os comandos git add e git commit:

git add arquivo.txt
git commit -m "Mensagem de commit"
